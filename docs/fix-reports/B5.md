# B5: profile age and gender prompts merged into one bubble

Status: FIXED
Branch: bugfix-loop4-timeline

## Symptom

On the profile beat, the coach's age prompt and gender prompt rendered as one
merged coach bubble ("Awesome {name}, two quick things so I can tailor this to
you. How old are you? And what's your gender?") instead of separate turns. The
flow-annotated reference (branch flow-annotated-render, beats/profile.tsx)
renders three separate coach turns: greeting, askAge, askGender.

## Root cause

Two layers conspired:

1. The transform (src/onboarding-flow/transform/designerToFlow.ts) marked the
   profile spec `openerFromEngine`, which IGNORED the designer's three separate
   props (greeting, askAge, askGender in designer-source.json) and substituted a
   single hand-written paragraph as the engine opener.
2. The renderer (BeatView) always turned `node.voice.openerText` into exactly
   one coach step, so even a correctly-authored multi-prompt opener had no way
   to render as multiple turns.

## Fix

1. Transform: `resolveOpener` now composes the profile opener from the
   designer's own props, joined with newlines. Convention established: a
   newline inside an opener is a TURN BREAK. The engine fallback opener for
   profile carries the same three-line shape. Regenerated
   flows/onboarding-beginner-v1.generated.json and
   src/generated/onboarding_combined.json via `npm run flow:sync` (no generated
   JSON hand-edits).
2. Renderer: new pure leaf `renderer/openerTurns.ts` splits an opener into turn
   lines. BeatView renders one coach step per line on the active beat (BeatPlayer
   already supported multi-step reveal), and one static bubble per line on past
   beats. PastBeatBubbles applies the same split.

## Verified

- Preview walk (http://localhost:5322/onboarding-flow-preview?startAt=profile):
  profile now renders three coach bubbles in sequence ("Good to meet you,
  there. Two quick things so I can tailor this to you." / "How old are you?" /
  "And your gender?"), card reveals after the last one.
- New tests: transform test asserts the three-line opener shape; openerTurns
  unit tests cover split/trim/empty cases. Full onboarding-flow suite green.

## Upstream prevention

The turn-break convention lives in the transform (documented at resolveOpener)
and in openerTurns.ts, and the transform test pins the profile opener to three
lines. Any future designer edit flows through `npm run flow:sync`; a regression
back to a merged paragraph fails the B5 transform test.
