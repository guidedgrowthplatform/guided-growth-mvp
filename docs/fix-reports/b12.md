# B12 fix report: LLM invents process talk ("confirm your path choice first", "I'll save that for you")

Branch: `bugfix-loop3-llm` | MR: !401 | Date: 2026-07-02

## Verdict: no additional prompt change needed on this branch

The plan's instruction was to verify the invented process talk after Loop 2's map fix and tighten only if something remains. Checked, nothing remains to tighten:

1. **The step-ladder cause is fixed on `bugfix-loop2-resume`.** Staging's `api/_lib/llm/onboarding/systemPromptAddendum.ts` still carries the stale ladder (`plan-review(7)->8, morning(8)->9, reflection(9)->10`), which made the model believe earlier steps (like the path choice) were unconfirmed and say so out loud. Loop 2's rewrite replaces it with the correct beat map plus a SELF-ADVANCING BEATS block and clarifies that step numbers are beat identities. That is the root of the "you need to confirm your path choice first" utterance. It lands with MR !398's branch; this loop must not duplicate it.

2. **The narration phrases themselves are already hard-banned in every onboarding prompt.** `NO_INTERNAL_NARRATION_RULE` (always included by `buildSystemPromptForRequest`) explicitly forbids "I'll", "I'm going to", announcing saves/writes/updates, and confirming tool success, which covers "I'll save that for you". `NO_PRENARRATION_RULE` bans narrating navigation and next-screen tasks. The tool addendum adds "Do NOT narrate the save" and "do NOT pre-narrate".

3. **Grep for inviting text came back clean.** No prompt source (systemPromptAddendum, beatContexts, noPrenarrationRule, noInternalNarrationRule, checkin addenda) instructs or implies the coach should narrate steps or tools. The only "confirmation" language left is the intentional revisit-switch line on the path fork ("ready to move on, or change anything?"), which is user-facing UX by design, not tool narration.

## What this loop did contribute against B12's symptom

The B11 client fix stops raw error codes and provider error text from rendering as coach bubbles, which was a second source of "the coach says machine things" on the call recordings.

## Residual risk / follow-up

If process talk still appears after !398 (Loop 2) and !401 (this branch) are both merged, the next lever is per-beat DO-NOT lines in `beatContexts.ts` (the beats-design handoff already defines that structure), not another global rule.
